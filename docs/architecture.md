# 아키텍처 설계서

## 1. 전체 구성도
- 이 프로젝트는 백엔드 API 중심으로 구현한 뒤 프론트 엔드와 db와 합칠 것이다.
- 외부 클라이언트(웹 앱, 모바일 앱, 테스트 도구 등)가 HTTP 요청을 보내고, 백엔드가 JSON 형태로 응답을 반환한다.
- 핵심 구성 요소는 다음과 같다.
  - API Server: NestJS
  - Auth & User Management: JWT 기반 인증
  - AI Service: Claude API
  - Database: PostgreSQL
  - Admin Data Management: 규정/일정/기업 정보 관리
  - Infra: AWS EC2 + RDS

## 2. 모듈 구성
- Backend API: 사용자 요청 처리, 인증, 데이터 조회, AI 호출
- Database Layer: 학사 규정, 일정, 기업 정보, 채팅 기록 저장
- AI Layer: 질문 의도 파악, 답변 생성, 정책 기반 응답 조합
- Admin Layer: 관리자용 데이터 등록/수정/삭제 기능
- Infra Layer: 환경 변수 관리, 로깅, 배포, 백업

## 3. 주요 흐름
1. 클라이언트가 질문 또는 요청 데이터를 백엔드 API로 전송한다.
2. 백엔드가 인증 및 권한을 확인한다.
3. 데이터베이스에서 관련 학사 정보와 규정 데이터를 조회한다.
4. 필요 시 AI 서비스에 프롬프트를 전달해 답변을 생성한다.
5. 백엔드가 구조화된 JSON 응답을 반환한다.

## 4. 보안
- JWT 기반 인증과 권한 검증을 사용한다.
- 개인정보는 최소 저장 원칙을 적용한다.
- 민감 정보는 마스킹 처리한다.
- 관리자 기능은 권한이 있는 사용자만 접근 가능해야 한다.
- API 키와 DB 접속 정보는 환경 변수로 관리한다.

## 5. 운영 포인트
- 요청 로그와 에러 로그를 기록한다.
- AI 응답 실패 시 대체 응답 또는 예외 처리 흐름을 둔다.
- 외부 API 장애 시 재시도 로직과 실패 메시지를 제공한다.
- 운영 환경에서는 모니터링과 백업 체계를 갖춘다.
